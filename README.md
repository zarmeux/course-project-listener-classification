# Классификация типов слушателей на основе активности в Last.fm

## О проекте

Проект посвящён анализу истории прослушиваний Last.fm и классификации пользователей по характеру их активности. Основной фокус сделан на временных паттернах прослушивания: распределении активности по дням, часам, частям суток и сессиям.

Проект реализован как воспроизводимая последовательность Jupyter-ноутбуков, которая:

- очищает исходные данные;
- строит временные признаки;
- формирует итоговые аналитические таблицы;
- рассчитывает пользовательские признаки;
- присваивает типы слушателей;
- готовит материалы для отчёта и загрузки в Yandex DataLens.

## Данные

Источник данных:

```text
data/raw/Last.fm_data.csv
```

Текущие параметры выборки:

- `166153` прослушивания;
- `11` пользователей;
- период наблюдения: `31` день.

После очистки и построения временных признаков формируется промежуточная таблица:

- `data/interim/lastfm_time_features.csv`

После этого в `data/processed/` сохраняются шесть итоговых таблиц для анализа и DataLens.

## Логика пайплайна

Последовательность обработки данных:

```text
raw data -> clean data -> time features -> activity datasets -> user features -> listener types -> report artifacts -> datalens
```

Основные этапы:

1. Очистка исходных данных и нормализация названий полей.
2. Построение временных признаков: дата, час, день недели, часть суток.
3. Формирование таблиц активности:
   - отдельные прослушивания;
   - дневная активность пользователей;
   - активность по сетке `день недели × час`;
   - сессии прослушивания.
4. Расчёт пользовательских признаков.
5. Rule-based классификация пользователей.
6. Построение локальных графиков и сводных таблиц.

## Классификация пользователей

### Основной тип слушателя

Поле `listener_type` принимает одно из трёх значений:

- `Интенсивный`
- `Регулярный`
- `Эпизодический`

Правила присвоения:

- `Интенсивный`, если `avg_daily_plays >= 75-го перцентиля`;
- `Эпизодический`, если `active_days_share <= 25-го перцентиля`;
- иначе `Регулярный`.

### Дополнительный временной профиль

Поле `behavior_profile` описывает характер распределения активности по частям суток.

Используются значения:

- `Ночная активность`
- `Утренняя активность`
- `Дневная активность`
- `Вечерняя активность`
- `Смешанная активность`

Правило присвоения:

- если разница между максимальной и второй по величине долей среди `morning_share`, `day_share`, `evening_share`, `night_share` меньше `0.10`, пользователю присваивается `Смешанная активность`;
- иначе профиль определяется по полю `main_day_part`.

## Порядок запуска ноутбуков

Основная логика проекта вынесена в пять ноутбуков из папки `notebooks/`.

Последовательность запуска:

1. `01_clean_and_time_features.ipynb`
2. `02_activity_datasets.ipynb`
3. `03_user_features.ipynb`
4. `04_listener_types.ipynb`
5. `05_report_and_checks.ipynb`

Назначение ноутбуков:

- `01_clean_and_time_features.ipynb` — очистка исходных данных и построение временных признаков;
- `02_activity_datasets.ipynb` — построение таблиц активности;
- `03_user_features.ipynb` — построение таблицы `user_features.csv`;
- `04_listener_types.ipynb` — присвоение типов слушателей и временных профилей;
- `05_report_and_checks.ipynb` — построение итоговых графиков, summary-таблиц и финальных проверок.

Ноутбуки должны запускаться сверху вниз с чистого ядра.
