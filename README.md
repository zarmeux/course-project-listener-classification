# Анализ и визуализация данных с использованием Yandex DataLens, на примере данных Last.FM

Репозиторий содержит материалы курсовой работы, посвящённой анализу истории прослушиваний Last.fm и классификации пользователей по поведенческим паттернам музыкальной активности.

В проекте рассматриваются различные характеристики пользовательского поведения: регулярность прослушивания, средняя интенсивность активности, распределение прослушиваний по дням, часам, частям суток, будням и выходным дням, а также структура сессий прослушивания.

## Цель проекта

Цель проекта — обработать историю прослушиваний Last.fm за январь 2021 года, выделить устойчивые поведенческие паттерны музыкальной активности пользователей и подготовить данные для аналитических визуализаций и интерактивного дашборда в Yandex DataLens.

В рамках работы решаются следующие задачи:

1. выполнить предварительную очистку исходного датасета;
2. построить временные признаки для каждого прослушивания;
3. сформировать итоговые таблицы активности пользователей;
4. рассчитать признаки пользователей;
5. реализовать rule-based классификацию типов слушателей;
6. построить локальные аналитические графики в Python;
7. подготовить итоговые CSV-файлы для загрузки в Yandex DataLens;
8. описать структуру интерактивного дашборда.

## Используемые данные

В проекте используется датасет истории прослушиваний Last.fm.

Источник данных:

```text
data/raw/Last.fm_data.csv
```

Ссылка на источник:

> Lasf.FM_dataset: https://www.kaggle.com/datasets/harshal19t/lastfm-dataset

Исходные поля датасета:

- `Username`
- `Artist`
- `Track`
- `Album`
- `Date`
- `Time`

Одна строка исходного датасета соответствует одному факту прослушивания музыкальной композиции пользователем.

Параметры выборки:

```text
Период анализа: 01.01.2021 - 31.01.2021
Количество прослушиваний: 166153
Количество пользователей: 11
```

После очистки данных и построения временных признаков формируется промежуточная таблица:

```text
data/interim/lastfm_time_features.csv
```

Итоговые таблицы для анализа и Yandex DataLens сохраняются в папку:

```text
data/processed/
```

## Основные этапы работы

### 1. Очистка данных и построение временных признаков

На первом этапе выполняются:

- удаление технических столбцов;
- проверка обязательных полей;
- приведение названий колонок к единому формату;
- очистка пустых строк;
- объединение `date` и `time` в `datetime`;
- удаление некорректных дат и дубликатов;
- построение временных признаков.

Итог этого этапа:

```text
data/interim/lastfm_time_features.csv
```

### 2. Построение таблиц активности

На втором этапе формируются:

- таблица отдельных прослушиваний;
- таблица дневной активности пользователей;
- таблица активности по сочетанию `день недели x час`;
- таблица сессий прослушивания.

### 3. Расчёт признаков пользователей

На третьем этапе рассчитываются новые показатели, описывающие активность пользователей:

- число прослушиваний;
- доля активных дней;
- среднее и медианное число прослушиваний;
- доли активности по частям суток;
- доли активности в будни и выходные;
- наиболее характерная часть суток;
- число и длительность сессий;
- длина серий активных и неактивных дней.

### 4. Классификация типов слушателей

На четвёртом этапе пользователям присваиваются:

- основной тип слушателя `listener_type`;
- дополнительный временной профиль `behavior_profile`.

Используются три основных типа:

- `Интенсивный`
- `Регулярный`
- `Эпизодический`

Дополнительные профили:

- `Ночная активность`
- `Утренняя активность`
- `Дневная активность`
- `Вечерняя активность`
- `Смешанная активность`

Одному пользователю может быть присвоен только один основной тип и один дополнительный тип.

### 5. Локальная аналитика и проверка результатов

На последнем этапе строятся:

- локальные графики в `report/figures/`;
- summary-таблицы в `report/tables/`;
- финальные проверки итоговых таблиц и показателей.

## Как запустить проект

### 1. Клонировать репозиторий

```bash
git clone https://github.com/zarmeux/course-project-listener-classification.git
cd course-project-listener-classification
```

### 2. Создать виртуальное окружение

Для macOS/Linux:

```bash
python -m venv .venv
source .venv/bin/activate
```

Для Windows:

```bash
python -m venv .venv
.venv\Scripts\activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Запустить ноутбуки

Ноутбуки рекомендуется запускать последовательно:

```text
01_clean_and_time_features.ipynb
02_activity_datasets.ipynb
03_user_features.ipynb
04_listener_types.ipynb
05_report_and_checks.ipynb
```

Порядок запуска важен, так как последующие ноутбуки используют таблицы, созданные на предыдущих этапах.

## Результаты

В результате работы были получены очищенные и подготовленные таблицы активности пользователей Last.fm, рассчитаны признаки пользовательского поведения и подготовлены визуализации для анализа временных паттернов музыкальной активности.

Основные содержательные результаты:

- историю прослушиваний можно описывать через регулярность, интенсивность и временную структуру активности;
