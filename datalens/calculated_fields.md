# Вычисляемые поля DataLens

## Назначение документа

Документ описывает вычисляемые поля, которые используются при построении дашборда в Yandex DataLens.

## Основные показатели

### Количество прослушиваний

Используется для KPI и графиков на основе таблицы `listenings_datalens.csv`.

```text
COUNT([listening_id])
```

### Количество пользователей

Используется для KPI и итоговых сводок.

```text
COUNTD([username])
```

### Количество активных дней

Используется для KPI по выборке дат.

```text
COUNTD([date_only])
```

### Количество сессий

Используется для KPI на основе таблицы `listening_sessions.csv`.

```text
COUNTD([session_id])
```

## Относительные показатели

### Доля прослушиваний

Используется для расчёта доли категории в общем объёме прослушиваний.

```text
SUM([plays_count]) / TOTAL(SUM([plays_count]))
```

## Служебные поля

### Порядок дней недели

Для корректной сортировки дней недели используется числовое поле `weekday_number`.

Соответствие значений:

```text
0 = Понедельник
1 = Вторник
2 = Среда
3 = Четверг
4 = Пятница
5 = Суббота
6 = Воскресенье
```

## Примечание

Часть полей подготовлена на этапе обработки данных и не требует повторного вычисления в DataLens. К таким полям относятся:

- `day_part`
- `weekday`
- `weekday_number`
- `month`
- `month_number`
- `is_weekend`
- `weekend`
- `listener_type`
- `behavior_profile`

Использование заранее подготовленных полей обеспечивает единообразие расчётов между Python-скриптами и дашбордом DataLens.
