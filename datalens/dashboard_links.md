# Таблицы и связи для Yandex DataLens

## Назначение документа

Документ фиксирует состав итоговых таблиц, подготовленных для загрузки в Yandex DataLens, а также схему их использования при построении дашборда.

## Итоговые таблицы

В папке `data/processed/` сформированы следующие таблицы:

| Файл | Размер | Содержание |
|---|---:|---|
| `listenings_datalens.csv` | `166153 x 13` | таблица прослушиваний |
| `user_daily_activity.csv` | `341 x 18` | дневная активность пользователей |
| `user_weekday_hour_profile.csv` | `1848 x 9` | активность пользователей по дням недели и часам |
| `listening_sessions.csv` | `1465 x 19` | таблица сессий прослушивания |
| `user_features.csv` | `11 x 27` | итоговые признаки пользователей |
| `user_activity_types.csv` | `11 x 18` | типы слушателей и дополнительные временные профили |

## Таблицы, загружаемые в DataLens

Для построения дашборда используются все шесть итоговых таблиц:

1. `listenings_datalens.csv`
2. `user_features.csv`
3. `user_activity_types.csv`
4. `user_daily_activity.csv`
5. `user_weekday_hour_profile.csv`
6. `listening_sessions.csv`

## Принцип организации данных в DataLens

Подготовленные таблицы описывают данные на разных уровнях:

- `listenings_datalens.csv` содержит отдельные прослушивания;
- `user_daily_activity.csv` содержит активность пользователя по дням;
- `user_weekday_hour_profile.csv` содержит активность пользователя по сочетанию `день недели × час`;
- `listening_sessions.csv` содержит сессии прослушивания;
- `user_features.csv` и `user_activity_types.csv` содержат одну строку на пользователя.

В связи с этим целесообразно использовать несколько отдельных датасетов DataLens, а не один общий датасет. Такое решение соответствует структуре исходных таблиц и позволяет избежать дублирования строк при объединении таблиц с разным числом наблюдений на одного пользователя.

## Ключи связи

Основным ключом связи выступает поле `username`.

Используются следующие связи:

- `listenings_datalens.username = user_features.username`
- `user_features.username = user_activity_types.username`
- `user_daily_activity.username = user_features.username`
- `user_weekday_hour_profile.username = user_features.username`
- `listening_sessions.username = user_features.username`

## Рекомендуемый состав датасетов DataLens

### 1. Датасет прослушиваний

Основная таблица:

- `listenings_datalens.csv`

Связанные таблицы:

- `user_features.csv`
- `user_activity_types.csv`

Назначение:

- показатели общего объёма прослушиваний;
- графики по датам;
- графики по часам;
- графики по дням недели;
- фильтрация по пользователю;
- разрезы по типу слушателя.

### 2. Датасет дневной активности

Основная таблица:

- `user_daily_activity.csv`

Связанные таблицы:

- `user_features.csv`
- `user_activity_types.csv`

Назначение:

- анализ активности пользователей по дням;
- сравнение пользователей по числу активных дней;
- сравнение пользователей по интенсивности прослушивания.

### 3. Датасет по дням недели и часам

Основная таблица:

- `user_weekday_hour_profile.csv`

Связанные таблицы:

- `user_features.csv`
- `user_activity_types.csv`

Назначение:

- построение heatmap `weekday × hour`;
- сравнение временных профилей пользователей;
- фильтрация по типу слушателя и дополнительному профилю.

### 4. Датасет сессий

Основная таблица:

- `listening_sessions.csv`

Связанные таблицы:

- `user_features.csv`
- `user_activity_types.csv`

Назначение:

- анализ числа сессий;
- анализ длительности сессий;
- сравнение сессий по типам слушателей.

### 5. Датасет пользователей

Основная таблица:

- `user_features.csv`

Связанная таблица:

- `user_activity_types.csv`

Назначение:

- scatter-графики по пользовательским признакам;
- распределение по `listener_type`;
- распределение по `behavior_profile`;
- таблица пользователей с итоговыми признаками.

## Вывод

На текущем этапе все итоговые таблицы для Yandex DataLens подготовлены. Схема связей определена через поле `username`. Для построения дашборда используется набор отдельных датасетов, соответствующих различным типам аналитических задач: анализу прослушиваний, дневной активности, временных профилей, сессий и пользовательских признаков.
