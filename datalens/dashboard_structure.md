# Структура дашборда DataLens

## Общая логика дашборда

Дашборд состоит из трёх страниц:

1. `Общий обзор`
2. `Временные паттерны`
3. `Типы слушателей`

Структура дашборда соответствует исследовательской задаче проекта: анализу пользовательской активности, временных паттернов прослушивания и итоговой классификации пользователей.

## Страница 1. Общий обзор

### Цель страницы

Показать объём данных и общую динамику прослушиваний.

### Основные таблицы

- `listenings_datalens.csv`
- `listening_sessions.csv`

### Элементы страницы

- KPI: всего прослушиваний
- KPI: количество пользователей
- KPI: количество активных дней
- KPI: количество сессий
- линейный график прослушиваний по датам
- столбчатый график прослушиваний по часам
- столбчатый график прослушиваний по дням недели

### Фильтры

- `username`
- `date_only`
- `day_part`
- `weekend`

### Аналитический смысл

Страница показывает общий масштаб выборки и позволяет оценить распределение активности по времени. На этой странице фиксируется объём наблюдений, число пользователей и общее распределение прослушиваний по датам, часам и дням недели.

## Страница 2. Временные паттерны

### Цель страницы

Показать различия во времени прослушивания между пользователями.

### Основные таблицы

- `listenings_datalens.csv`
- `user_daily_activity.csv`
- `user_weekday_hour_profile.csv`

### Элементы страницы

- распределение прослушиваний по частям суток
- heatmap `weekday x hour`
- сравнение будней и выходных
- график активности пользователей по дням
- график или таблица по `active_days` и `avg_daily_plays`

### Фильтры

- `username`
- `day_part`
- `weekday`
- `weekend`

### Аналитический смысл

Страница показывает, что активность пользователей распределена по времени неравномерно. С её помощью можно выявить вечернюю, ночную и выходную активность, а также сравнить пользователей по регулярности и интенсивности прослушивания.

## Страница 3. Типы слушателей

### Цель страницы

Показать итоговую классификацию пользователей по поведенческим признакам.

### Основные таблицы

- `user_features.csv`
- `user_activity_types.csv`

### Элементы страницы

- распределение пользователей по `listener_type`
- распределение пользователей по `behavior_profile`
- scatter plot: `active_days_share x avg_daily_plays`
- scatter plot: `night_share x weekend_share`
- таблица пользователей со следующими полями:
  - `username`
  - `listener_type`
  - `behavior_profile`
  - `active_days_share`
  - `avg_daily_plays`
  - `night_share`
  - `evening_share`
  - `weekend_share`
  - `session_count`
  - `avg_session_length`

### Фильтры

- `listener_type`
- `behavior_profile`
- `username`

### Аналитический смысл

Страница отражает итоговую интерпретацию данных. Она показывает, как пользователи распределяются по основным типам слушателей и по дополнительным временным профилям, а также позволяет сравнить их по ключевым числовым признакам.

## Итог

Предлагаемая структура дашборда охватывает три уровня анализа:

- общий масштаб данных;
- временные паттерны пользовательской активности;
- итоговую классификацию пользователей.

Такая структура соответствует составу подготовленных таблиц и позволяет последовательно перейти от общего обзора данных к интерпретации типов слушателей.
